# 戒烟助手 App 原型 v2

## 目标

基于现有 PWA 功能，把界面从“功能堆叠”收敛成一套更像正式产品的移动端原型。整体方向是轻医疗、日常陪伴、危机干预：可信但不冰冷，鼓励但不鸡血。

## 信息架构

1. 首次设置
   - 戒烟开始时间
   - 每日吸烟量
   - 每包价格、每包支数

2. 主页
   - 戒烟总时长
   - 焦油减少、尼古丁减少、一氧化碳、节省金额、少抽香烟、延长寿命
   - 下一健康里程碑
   - 每日鼓励语
   - SOS 呼吸入口

3. 身体恢复
   - 器官恢复地图
   - 器官当前恢复百分比
   - 当前阶段标题
   - 具体改善收益
   - 男性专属好处、湿热体质改善折叠卡

4. 成就
   - 时间类成就
   - 省钱类成就
   - 已解锁/未解锁状态

5. 设置
   - 戒烟日期
   - 每日吸烟量
   - 每包价格
   - 每包支数
   - 保存、重置、添加到主屏幕提示

6. SOS 呼吸
   - 4-7-8 呼吸流程
   - 第 1/3 轮进度
   - 完成反馈

## 原型画面

建议在 Figma 中做 8 个移动端画板，尺寸 `390 x 844`：

1. `00 原型说明`
   - 产品定位、视觉方向、信息架构、主流程。

2. `01 首次设置 - 戒烟时间`
   - 中央大图标和问题：“从哪一刻开始？”
   - 日期选择输入框。
   - 主按钮：“下一步”。
   - 底部步骤进度。

3. `02 首次设置 - 吸烟习惯`
   - 三张参数卡：每日吸烟量、每包价格、每包支数。
   - 主按钮：“开始我的戒烟之旅”。

4. `03 主页 - 每日总览`
   - 顶部轻量日期/第几天标签。
   - 大型戒烟天数卡，强调“45 天 06 时 23 分”。
   - 2 行 x 3 列数据卡。
   - 下一里程碑卡。
   - 每日鼓励语卡。
   - 红色 SOS 呼吸按钮。
   - 底部导航。

5. `04 身体恢复 - 器官地图`
   - 顶部当前器官详情卡。
   - 中央器官恢复地图。
   - 底部收益列表。
   - 折叠卡入口放在下方滚动区域。

6. `05 成就 - 徽章墙`
   - 顶部总览：“已解锁 8 / 14”。
   - 3 列徽章网格。
   - 未解锁状态降低透明度、去饱和。

7. `06 设置 - 参数维护`
   - App 信息区。
   - 分组设置列表。
   - 保存按钮、重置按钮。
   - PWA 安装提示。

8. `07 SOS - 呼吸练习`
   - 全屏沉浸式界面。
   - 中央呼吸圆，显示阶段和倒计时。
   - 轮次提示。
   - 退出按钮。
   - 完成态可作为同画板变体或额外画板。

## 交互流

1. 首次进入：
   - `01 首次设置 - 戒烟时间` -> `02 首次设置 - 吸烟习惯` -> `03 主页 - 每日总览`

2. 主导航：
   - `03 主页` <-> `04 身体恢复` <-> `05 成就` <-> `06 设置`

3. 危机干预：
   - `03 主页 - SOS 按钮` -> `07 SOS - 呼吸练习`
   - 完成后返回 `03 主页`

4. 身体恢复：
   - 点击不同器官，更新顶部详情卡、恢复百分比、收益列表。

## 需要重点改善的地方

1. 信息层级
   - 主页只保留一个最强视觉焦点：戒烟天数。
   - 数据卡统一尺寸和颜色逻辑，避免每个模块都在抢注意力。

2. 视觉风格
   - 减少强渐变和装饰光斑。
   - 用白色卡片、浅绿色背景、克制阴影来表达健康和可信。

3. SOS 入口
   - 红色只用于烟瘾危机场景。
   - 按钮文案从“我要抽烟了”改成更行动导向：“烟瘾来了：开始 4-7-8 呼吸”。

4. 身体恢复图
   - 当前器官详情卡要比地图更容易读。
   - 地图负责探索，详情卡负责解释。

5. 成就页
   - 增加总览进度，避免只是徽章列表。
   - 未解锁状态保持可读，但降低饱和度。
